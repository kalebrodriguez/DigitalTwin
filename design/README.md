# Design references

Design-system docs from [meliwat/awesome-ios-design-md](https://github.com/meliwat/awesome-ios-design-md)
(MIT) that inform DigitalTwin's app design. We keep our own brand
(#5CA3FF on white) and borrow production patterns, not palettes:

- **`calm-DESIGN-expo.md`** → patient mode: haptic feedback conventions
  (soft impact on press, success notification on completion), pressed-state
  color + scale on primary buttons, serif brand voice for greetings.
- **`life360-DESIGN-expo.md`** → caregiver mode: member-row anatomy
  (ringed avatar, status line with icon), the battery glyph as a safety
  signal, notification bell with alert dot, and rationed semantic colors
  (safe green / alert red reserved for meaning).
